> [!IMPORTANT]
> This project is connected to [Lovable](https://lovable.dev). Avoid rewriting
> published git history — force pushing, or rebasing/amending/squashing commits
> that are already pushed — as it rewrites history on Lovable's side and the
> user will likely lose their project history.
>
> Commits you push to the connected branch sync back to Lovable and show up in
> the editor, so keep the branch in a working state.

## Development Rules

> [!IMPORTANT]
> **Planning Before Coding**: Always write an implementation plan and ask the user for confirmation or if they have edits to the plan *before* writing any code. After you create a plan, stop for a while and don't write or execute anything else until you review it with the user. Never skip this planning phase unless explicitly bypassed by the user.
